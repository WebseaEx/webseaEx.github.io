    現在の計画を照会する

    アドレス: /openApi/contract/currentTactics

    受け取るパラメーター

    必要

    提出方法:get

    提出パラメーター

    パラメータ名 必須かどうか タイプ 例 説明
    symbol 否 string EOS-USDT 取引ペア(不要の場合は全部)
    from 否 int 122 クエリの開始order_id
    direct 否 string prev クエリ方向(デフォルト prev)、prev 前方へ、時間（またはID）の降順；next 後方へ、時間（またはID）の昇順。（例えば一列の数：1、2、3、4、5。from=4 の場合、prev は3、2、1；next は5だけ）
    limit 否 int 20 ページごとの結果の数の上限で、デフォルトは20、最大で100（詳細はページネーションの説明を参照してください）
    is_full 否 int 1 1逐倉 2全倉

    返却パラメーター

    パラメータ名 必要かどうか タイプ 例 説明
    errno は int 0 戻り状態コード
    errmsg 是 string 成功 返信情報
    result は object 例を参照 戻り値データ

    resultパラメーター

    パラメータ名 必要かどうか 型 例 説明
    order_id はい string BG5000181583375122413SZXEIX 注文番号
    ctime はい int 1576746253 委託時間
    symbol は string ETH-USDT 取引ペア
    price はい string 1 委託価格
    lever_rate はい int 10 レバレッジ倍率
    amount はい string 10 委託数量
    type はい string buy-limit 委託タイプ
    trigger_status はい int 0 トリガー状態（0未触発/1発動）
    trigger_sign はい string 0 トリガー条件（1は以上／2は以下）
    trigger_price はい string 10 トリガー価格
    stop_profit_price は string 15 利益確定の価格
    stop_loss_price はい string 10 損失を停止する価格

    返却例

    JSON
    {
    "errno": 0,
    "errmsg": "success",
    "result": [
        {
            "order_id": "BG5000181583375122413SZXEIX", //注文番号
            "ctime": 1576746253, //委託時間
            "symbol": "ETH-USDT", //取引ペア
            "price": "1", //委託価
            "lever_rate": 10, //レバレッジ倍数
            "amount": "10", //委託数
            "type": "buy-limit", //委託タイプ
            "trigger_status": 0, //トリガー状態 0未トリガー 1トリガー済み
            "trigger_sign": "0", //トリガー条件 1以上 2以下
            "trigger_price": "10", //トリガー価格
            "stop_profit_price": "15", //利益確定の価格
            "stop_loss_price": "10" //損切り価格
        }
    ]
}
